Fixed Scroll

Less laggy and annoying than your average autoscrolls

This really is just a place holder to take up a little more "header space."

Instructions: I am sick and tired of javascript based fixed positioning. It's laggy at best and broken at work. Fortunately, there is CSS's fixed positioning, but that doesn't always work perfectly either if there are times when it shouldn't be fixed. So, this attempt is a pretty straight forward way to fix it. The box stays in a relative position until the page scrolls to far (past where it needs to be for a fixed position. It then just animates to the new correct position and switches over to fixed positioning. When it hits the next gate it stops.

A little info about the algorithm. It's pretty basic as a rule. The scroll position of a reference object (in this case the window) determines how far along a path you are. The scrolling objects position is just a function of the scroll position of the reference object. (No idea if that makes sense to anyone else.)
